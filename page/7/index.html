macos下LAMP、LNMP php环境搭建

发表于 2020-08-22

当然是首选图形界面的，想过宝塔 但是要用docker
有点折腾了。轻量开发而已。。懒得搞。于是决定用自带的apache和php，也放弃了nginx。
省的折腾，反正就是一个轻量开发环境。apache和nginx对我来说无差

最后我还是换了虚拟机装了宝塔，Docker宝塔很不方便，占用硬盘也不小。暂时用的pd，回头有时间换Vmware

1、准备和安装
1.1、查看apache和php版本

我的是 Apache/2.4.33 (Unix) PHP/7.1.16 macos10.13.6

1.2、安装mysql
，有好多版本没有macos版，本来打算安装5.5的懒得找，随便找了一个5.7
https://downloads.mysql.com/archives/get/p/23/file/mysql-5.7.9-osx10.10-x86_64.dmg
安装完成会提示 默认密码
2020-03-10T06:30:11.847539Z 1 [Note] A temporary password is generated
for root@localhost: )W_w1:=j>trV
安装后系统》偏好设置，启动
cd /usr/local/mysql/bin/
./mysql -uroot -p
修改密码
set password for root@localhost = password('123');
其他版本从 https://downloads.mysql.com/archives/community/ 下载
2、修改apache支持php
sudo code /etc/apache2/httpd.conf
如果找不到code可以用vi、edit或者在vscode中shift + command + p
输入shell选择安装code到shell
找到 #LoadModule php7_module libexec/apache2/libphp7.so
去掉注释
3、修改apache默认路径
找到
DocumentRoot "/Library/WebServer/Documents"
<Directory "/Library/WebServer/Documents">
修改为 /Users/yanhui/www/htdocs 保存
sudo mkdir -p /Users/yanhui/www/htdocs
sudo chown -R _www:_www /Users/yanhui/www/htdocs
sudo chmod -R 777 /Users/yanhui/www/htdocs
创建一个文件
echo "<?php phpinfo(); ?>" >> /Users/yanhui/www/htdocs/index.php
下载phpmyadmin
4、启动和测试
开启Apache：sudo apachectl start
关闭Apache：sudo apachectl stop
重启Apache：sudo apachectl restart

linux下自动同步备份ftp空间

发表于 2020-08-22

可以在路由器或者vps中运行，实现自动备份网站ftp文件。
直接使用wget<!--more-->

mkdir /home/wwwroot/from_aliyun_web
cd /home/wwwroot/from_aliyun_web
#删除10天之前的备份
find /home/wwwroot/from_aliyun_web -mtime +10 -name "bak-from*" -exec rm -rf (花括号)(花括号) ;
rm -rm ./*.sql
#下载
wget -nH -m --ftp-user=  --ftp-password= ftp://.ftp.aliapp.com/htdocs/ 
#打包
tar -zcvf bak-from-aliyun-web-$(date +%Y%m%d-%H%M).tar.gz htdocs/
#mysqldump 没装的话，需要yum install mysql
mysqldump -P 3306 -h rm.mysql.rds.aliyuncs.com -u[root] -p[passwd] [table]>/home/wwwroot/from_aliyun_web/mysql-.sql
#打包
tar -zcvf bak-aliyun-mysql-.sql-$(date +%Y%m%d-%H%M).tar.gz mysql-.sql

chmod +x /root/auto/ftp_get.sh
crontab -e
01 02 * /root/auto/ftp_get.sh

iwork7恢复官方win8.1，解决声音和卡顿等问题

发表于 2020-08-22

手里有2个iwork7  1G 16G单系统版。
平时用来连蓝牙音响听听歌，玩个小游戏还可以，比较方便。功率小，可以用音响的U盘接口充电，24小时开机。之前淘宝买过两根OTG+HUB+充电一体的线
也比较方便，U盘可以一直挂在上面

安装过MSDN win8.1和win10  不是卡顿
就是死机，驱动从网络上找到了，但是声音一直不好解决。所以干脆重新刷了官方版本。但是官方给的镜像》4G 
fat32各种限制。最后找到一个32位的pe总算安装好了。安卓没玩过，没啥兴趣，手里一堆安卓平板，比如手机方便了哦。所以还是玩windows。win10搞不定
就搞win8了哦<!--more-->

现在把工具记录一下，准备把整个硬盘给ghost出来。没有恢复分区，只有 ESP
EFI等几个分区，C盘占用5G

 

ghost文件保存到云盘，方便以后使用。

没精简，只是激活了一下，删除一些自带的应用，修改了一下浏览器主页为百度，用dism++清理了一下垃圾，关闭了Windows
Defence 防火墙
自动更新等，UAC改为不降低亮度。自动更新服务也禁用了。有需要自己打开哦。

集成了winrar版  Dism++  以及 微PE工具箱

适用于iwork7  1G 
16G版本，注意，这是整个硬盘的镜像，可以直接恢复到硬盘上。不是分区的，要恢复到分区，请自行提取。

首选准备一个OTG线和usb hub。接好鼠标键盘 和U盘。

如果你的平板还可以启动，就直接安装一下
微PE32版，然后把下载的gho文件解压到U盘，然后重启进pe自带的一个CGI什么的ghost工具，图标在桌面上，进行硬盘恢复（切记不是分区恢复，不是用桌面上一个ghost恢复工具，用CGI的那个）就ok。如果不能启动
就找一个能启动的电脑 用微PE做一个32位的启动U盘，iwork7启动的时候 按F7
或者DEL 选u盘启动就好了  剩下的就一样了。

火狐浏览器任务栏两个图标的解决方法

发表于 2020-08-22

绿色版的火狐浏览器经常出现固定到任务栏的时候 会出现两个图标。
有大神写的批处理文件，运行一下，重新固定就ok了。膜拜,有洁癖的同学赶紧的吧

@echo off
title 解决火狐任务栏双图标——鱼刺
mode con cols=50 lines=25
color 0a
:-------------------------------------
REM --> Check for permissions
>nul 2>&1 "%SYSTEMROOT%system32cacls.exe" "%SYSTEMROOT%system32configsystem"
REM --> If error flag set, we do not have admin.
if '%errorlevel%' NEQ '0' (
echo Requesting administrative privileges...
goto UACPrompt
) else ( goto gotAdmin )
:UACPrompt
echo Set UAC = CreateObject^("Shell.Application"^) > "%temp%getadmin.vbs"
echo UAC.ShellExecute "%~s0", "", "", "runas", 1 >> "%temp%getadmin.vbs"
"%temp%getadmin.vbs"
exit /B
:gotAdmin
if exist "%temp%getadmin.vbs" ( del "%temp%getadmin.vbs" )
pushd "%CD%"
CD /D "%~dp0"
:--------------------------------------
goto MENU
:menu
cls
echo.
echo 解决火狐任务栏双图标——鱼刺
echo =================================================
echo.
echo 执行操作成功后关闭浏览器重新固定到任务栏
echo.
echo 1，执行此操作
echo.
echo 2，还原操作
echo.
echo 3，退出
echo.
echo =================================================
echo.
echo 请输入选择项目的序号,按回车键确认：
set /p ID=
if "%id%"=="1" goto cmd1
if "%id%"=="2" goto cmd2
IF "%id%"=="3" exit
PAUSE

:cmd1
echo 删除TaskBarIDs项，该项已备份在当前目录 bak.reg
reg export HKLMSOFTWAREWow6432NodeMozillaFirefoxTaskBarIDs bak1.reg
reg delete HKLMSOFTWAREWow6432NodeMozillaFirefoxTaskBarIDs /f
reg export HKCUSOFTWAREMozillaFirefoxTaskBarIDs bak2.reg
reg delete HKCUSOFTWAREMozillaFirefoxTaskBarIDs /f
PAUSE
goto MENU

:cmd2
echo 还原注册表
reg import bak1.reg
reg import bak2.reg
PAUSE
goto MENU
