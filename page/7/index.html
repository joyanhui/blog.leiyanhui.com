Dell venue 11 pro 7130 超频 v11p温度墙功耗墙

发表于 2020-08-22

古董机器了。。。一直用，最近卡的比较离谱

网上大概搜索了一下，原来是dell和intel给板子加了一个温度墙和功耗墙。根据网上资料多次尝试\

准备：usb-hub  鼠标键盘  U盘一个
软件：最新版本bois。为了安全期间用更新了A15的bois
https://www.dell.com/support/home/cn/zh/cndhs1/Drivers/DriversDetails?driverId=DVM7Y
DiskGenius
bootx64.efi
Intel(R) Extreme Tuning Utility

首先当然是刷bois到A15，据说A18也可以，未经测试
1、插入U盘删除所有分区，然后转换为GUID分区格式，创建esp分区100M就好了，去掉创建MSR。默认会多创建一个分区
2、在多创建的分区里面建立文件夹efi/boot 把bootx64.efi复制进来
3、重启电脑，点F12，进入bois setup
4、选择 boot sequence 再点击右边的add boot
option，点FileName后面的浏览（三个点），找到刚刚的bootx64.efi 点ok
5、在Boot Option Name里面随便输入几个字母，例如 USB boot For
efi，保存重启
6、再点F12选择 USB boot For efi 启动
7、顺利话进入efi的命令了。依次输入
setup_var 0xD0E 0x00 setup_var 0x54 0x00 setup_var 0xD04 0x00 setup_var 0xD0D 0x00 setup_var 0x39 0x00 setup_var 0x23 0x07 setup_var 0x3A 0x10
大致意思是 设置 地址 值。
setup_var 0x23 0x07 的意思是93度，同样84度就是 setup_var 0x23 0x10
默认温度是68 + 0x10【十进制16】 = 84
默认温度是68 + 0x16【十进制22】 = 90
默认温度是68 + 0x1b【十进制27】 = 95
cpu不能长时间运行在85以上的，所建议设置84度setup_var 0x23 0x10

然后重启电脑，安装Intel(R) Extreme Tuning Utility在Intel(R) Extreme
Tuning
Utility的左下角有一个温度的显示的勾号，鼠标移上去如果显示0-68就是没搞定，检查一下bois版本还有那一堆字符有没有输入错误。如果输错了，就再搞一边哦
然后 Intel(R) Extreme Tuning Utility里面的all
Control里面看看cpu功率位置默认6瓦
是不是也可一选择更多了。不过建议10瓦以内哦。

css html 在打印效果中的布局

发表于 2020-08-22

根据网上的资料，得知在网页打印中 屏幕的dpi不是传说中的72dpi 而是96dpi\

我们得知A4纸张的尺寸是210×297mm，按1英寸=25.41mm换算，即8.264×11.688英寸
所以，A4纸96dpi下的分辨率是794×1123。
也有一种说法是width:794px;height:1090px; 可能考虑到了页眉页脚

在票据打印机的时候，我使用no-webkit布局。页边距0.

centos挂载硬盘

发表于 2020-08-22

1 查看硬盘

fdisk -l

/dev/sda 就是需要挂载的硬盘了

fdisk /dev/sda

依次输入 n p 1 p w 创建一个分区
fdisk -l 再次查看，看到sda1

mkfs.ext4 /dev/sda1 #格式化

挂载

mkdir /data
mount /dev/sda1 /data

开机启动挂载

echo "/dev/sda1  /data  ext4    defaults        0 0" >> /etc/fstab

高恪路由器配置安装frp方法，服务端，最后改为ssh

发表于 2020-08-22

很幸福，鱼房的路由器有公网ip虽然不是固定的，总算可玩性高了很多。
网上大致看了一下说 高恪其实内置frp的 那么就找出来
看了一下是客户端，并没有frps。
自己电脑先下载 https://github.com/fatedier/frp/releases
猜测是_mips版本了，下载回来一看10m，路由器总共就16m的flash
我了个去，放弃
研究ssh穿透中

首先修改标准的liunux里面需要把
sshd_config里面GatewayPorts为yes不然路由器只能监听127.0.0.1
高恪是基于openwrt的没找到。。。。。暂时先用127.0.0.1了等回家刷一下路由

在没有端口的和内网地址的电脑上运行
就是把路由器ddns.3322.org的7280映射到本地80
ssh -fCNR 7280:localhost:80 root@ddns.3322.org -p22
就是把本地80端口和有外网的7280端口绑定一起

win下使用putty。设置 connection>ssh>Tunnels
填写
source 7280
Destination localhost:80
选择Remote 和 auto
进入Connection设置，Seconds between keeplives(0 to turn off)设置为30
防止掉线

添加 add登录后 查看7280端口
netstat -an

ssh换成autossh 据说更稳定，win下测试后再说
